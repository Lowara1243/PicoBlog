# Спецификации проекта PicoBlog

## Общая цель
Создать альтернативу WriteFreely, сфокусированную на производительности и простоте, без функций федерации.

## Ключевые особенности

### 1. Доступ и управление пользователями
*   **Регистрация и права доступа:** Только администратор может регистрировать новых пользователей и назначать им права.
*   **Контроль доступа к чтению:** Администратор может указывать теги; посты с этими тегами доступны только авторизованным пользователям. Пользователи будут видеть только те посты, теги которых им разрешены.

### 2. Загрузка контента
*   Возможность загружать контент (текст, изображения) напрямую из браузера, без необходимости в дополнительных приложениях.

### 3. Производительность
*   Приложение должно быть достаточно производительным для работы на Raspberry Pi, учитывая, что контент будет состоять преимущественно из текста и изображений.

### 4. Создание контента
*   Быстрое создание контента с использованием Markdown.
*   Рендеринг Markdown в HTML/CSS на стороне сервера.

### 5. Современный и минималистичный дизайн
*   Современный, но минималистичный внешний вид без использования тяжелых фреймворков (например, React).
*   Предпочтение отдается TailwindCSS для стилизации.
*   Страницы просмотра блога должны содержать минимальное количество JavaScript (в идеале — ноль). JavaScript разрешен для страниц входа и административных интерфейсов.

### 6. Комментарии
*   Возможность оставлять комментарии к постам.
*   Поддержка Markdown в комментариях (с рендерингом на стороне сервера и обязательной санитарной обработкой).

### 7. Технические детали
*   **База данных:** SQLite для хранения данных (например, информация о постах, пользователях, тегах, комментариях).
*   **Рендеринг Markdown:** Выполняется на стороне сервера как для постов, так и для комментариев.
*   **Веб-сервер:** Nginx в качестве обратного прокси, особенно для обеспечения HTTPS.
*   **Фильтрация постов:** Возможность фильтровать посты на бэкенде на основе тегов, доступных пользователю.
